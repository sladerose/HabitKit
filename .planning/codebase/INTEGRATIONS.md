# External Integrations

**Analysis Date:** 2026-05-27

## APIs & External Services

**Database (primary):**
- Supabase PostgreSQL — hosted relational database
  - Client in use: `postgres` npm driver via Drizzle ORM (`src/db/index.ts`)
  - Connection: `DATABASE_URL` env var (session-mode pooler URL from Supabase)
  - Note: `@supabase/supabase-js` is installed but not imported in any source file; all DB access uses the direct `postgres` driver through Drizzle

**Supabase JS SDK (installed, not yet wired):**
- `@supabase/supabase-js` 2.106.2 is a declared dependency
- No Supabase client is instantiated in source; the SDK is unused at this time
- Likely intended for future auth or realtime features

## Data Storage

**Databases:**
- Type: PostgreSQL (hosted on Supabase)
- Connection var: `DATABASE_URL` (format: `postgresql://postgres:[password]@db.[project-ref].supabase.co:5432/postgres`)
- Client: `postgres` (node-postgres compatible driver) wrapped by `drizzle-orm/postgres-js`
- Schema managed by drizzle-kit; migrations output to `drizzle/`

## Environment Configuration

**Required env vars (from `.env.local.example`):**

| Variable | Purpose |
|----------|---------|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase project URL (exposed to browser) |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase anon/public key (exposed to browser) |
| `DATABASE_URL` | Direct PostgreSQL connection string for server-side Drizzle access |

**Secrets location:**
- `.env.local` — gitignored, loaded automatically by Next.js for server-side vars
- `drizzle.config.ts` reads `.env.local` manually via `fs.readFileSync` so drizzle-kit CLI has access

**Note on `NEXT_PUBLIC_` vars:**
- `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY` are declared in the example but the Supabase JS client is not instantiated anywhere in the codebase. These vars are currently unused at runtime.
