## Directory Purposes

**`src/app/`:**
- Purpose: Next.js App Router. Every `page.tsx` is a route. Every `route.ts` is an API endpoint.
- Contains: Server components (default), client components (`"use client"`), API handlers.
- Key files: `layout.tsx` (root shell), `page.tsx` (/ route)

**`src/app/api/`:**
- Purpose: REST API consumed by client components for mutations.
- Contains: Route handlers using `NextResponse`. No business logic — all delegation to `src/db/queries.ts`.
- Key files: `habits/route.ts`, `habits/[id]/route.ts`, `logs/route.ts`

**`src/components/`:**
- Purpose: Reusable UI components. All are `"use client"` — they handle interaction or use browser APIs.
- Contains: Interactive components only. No pure-presentational server components exist yet.
- Key files: `HabitCheckbox.tsx`, `SortableHabitList.tsx`, `HeatmapGrid.tsx`, `Navigation.tsx`

**`src/db/`:**
- Purpose: All database concerns in one place: schema, client, queries, seed data.
- Contains: Drizzle schema definitions, singleton DB client, query functions, seed script.
- Key files: `schema.ts` (source of truth for data shape), `queries.ts` (only import for DB access)

**`src/lib/`:**
- Purpose: Pure utility functions with no framework dependencies.
- Contains: Date arithmetic, streak algorithms, shared constants (colors, icons).
- Key files: `utils.ts`

**`drizzle/`:**
- Purpose: Migration SQL files generated by `drizzle-kit generate`.
- Generated: Yes (by `npm run db:generate`)
- Committed: Yes

**`public/`:**
- Purpose: Static files served at root. Includes PWA icons and SVG assets.
- Contains: `icon-192.png`, `icon-512.png` (PWA), `manifest.json`, SVG decoratives.

## Key File Locations

**Entry Points:**
- `src/app/layout.tsx`: Root HTML shell, loads Navigation, registers service worker
- `src/app/page.tsx`: Default route `/`, Today view

**Configuration:**
- `drizzle.config.ts`: Schema path, dialect, DB credentials for drizzle-kit CLI
- `tsconfig.json`: Path alias `@/*` -> `./src/*`
- `next.config.ts`: No active overrides

**Core Logic:**
- `src/db/schema.ts`: Canonical data model — `habits` and `habit_logs` tables
- `src/db/queries.ts`: All DB interaction functions; import this, never `src/db/index.ts` directly
- `src/lib/utils.ts`: Streak and completion logic; `HABIT_COLORS` and `HABIT_ICONS` constants

**Testing:**
- Not present. No test files or test framework configured.

## Naming Conventions

**Files:**
- Route files: lowercase (`page.tsx`, `route.ts`, `layout.tsx`) — Next.js convention
- Components: PascalCase (`HabitCheckbox.tsx`, `SortableHabitList.tsx`)
- DB files: lowercase (`schema.ts`, `queries.ts`, `index.ts`)
- Utilities: lowercase (`utils.ts`)

**Directories:**
- Routes: lowercase, hyphenated for multi-word (Next.js convention)
- Dynamic segments: bracket notation (`[id]`)

## Where to Add New Code

**New server-rendered page:**
- Create `src/app/<route>/page.tsx` as an async function (no `"use client"`)
- Add `export const dynamic = "force-dynamic"` at top
- Fetch directly from `src/db/queries.ts`

**New client-interactive page:**
- Create `src/app/<route>/page.tsx` with `"use client"` at top
- Use `fetch()` against existing API routes for data

**New reusable component:**
- Place in `src/components/`
- If it uses hooks or browser APIs, add `"use client"` at top
- If it is purely presentational (no interactivity), omit `"use client"` to allow server rendering

**New API endpoint:**
- Create `src/app/api/<resource>/route.ts`
- Export named functions `GET`, `POST`, `PUT`, `PATCH`, `DELETE` as needed
- Delegate all DB work to `src/db/queries.ts`

**New DB query:**
- Add a named export to `src/db/queries.ts`
- Import only from `src/db/index.ts` (the `db` instance) and `src/db/schema.ts`

**New DB table:**
- Add table definition to `src/db/schema.ts`
- Run `npm run db:generate` then `npm run db:migrate` (or `db:push` for dev)

**New utility function:**
- Add to `src/lib/utils.ts` if it is a pure function with no framework dependency
- Keep framework-specific helpers co-located with the feature that uses them
